# `j2` — Jinja2 templates

| Function | Description |
| --- | --- |
| `j2::template(template_text, data)` | Render a Jinja2 (minijinja) template with default options, using `data` as the context. Returns the rendered text. |
| `j2::engine()` | Start a configurable engine (see setters below); finish with `.template(template_text, data)`. |

```rhai
let text = j2::template(
  "Hello {{ name }}!{% for i in items %} {{ i }}{% endfor %}",
  #{ name: "world", items: [1, 2, 3] },
);
fs::write("/out.txt", text);
```

The context `data` accepts any Rhai value (maps, arrays, scalars).

`j2::engine()` returns a handle with chainable `with_*` setters (each
mutates the handle and returns it — see
[Method naming conventions](../conventions.md)), then renders with
`.template(...)`:

| Method | Description |
| --- | --- |
| `.with_undefined("lenient" \| "chainable" \| "strict")` | How undefined variables behave (default `"lenient"`; `"strict"` raises an error). |
| `.with_trim_blocks(bool)` | Strip the first newline after a block tag. |
| `.with_lstrip_blocks(bool)` | Strip leading whitespace before a block tag. |
| `.with_keep_trailing_newline(bool)` | Keep a single trailing newline (default strips it). |
| `.with_syntax(bs, be, vs, ve, cs, ce)` | Custom delimiters: block start/end, variable start/end, comment start/end. |
| `.with_include_root(path)` | Enable `{% include %}`/`{% extends %}`, loading templates from the given VFS directory. |

Includes are disabled unless `.with_include_root(path)` is set, and
loading is restricted to the VFS root (`..` escapes are rejected). Like
`fs::read`, includes reflect current disk state, not buffered writes.

```rhai
let text = j2::engine()
  .with_undefined("strict")
  .with_include_root("/templates")
  .template("{% include \"layout.txt\" %}", #{ body: "Hi" });
```
